 # Habits-MVPA

This repository contains the code, notebooks, and data used to reproduce analyses
and figures for the Habits-MVPA project. The repository is organized to make
analysis and figure generation reproducible from raw-ish CSVs and per-subject
trial files.

## Contents (high level)

- `data/` — raw and derived CSVs and per-subject trial CSVs.
   - `data/csvs/` contains project-level CSVs used by notebooks and scripts (e.g., `behavior.csv`, `rate.csv`).
   - `data/trial_info/` contains one row-per-trial CSVs named like `R01_0001-trial_info.csv` (the core per-subject event logs).
- `notebooks/` — Jupyter notebooks for figures and exploratory analyses. Executed versions are written to `notebooks/executed/`.
- `scripts/` — Python helper modules and small shell scripts used to load data, compute metrics, and run notebooks.
   - `scripts/util.py` — central utilities for reading trial files, computing consumption scores, devaluation ratios, block/run rates, and project-level path constants.
   - `scripts/imm.py` — IMM algorithm fitting utilities (annotated, typed).
   - `scripts/subjects.py` — subject lists and validation helpers.
   - `scripts/make_figures.sh` — shell script that executes notebooks and writes executed notebooks to `notebooks/executed/`.
- `results/` — analysis outputs, neuroimaging files (NIfTI), MVPA outputs, ROIs, and other produced artifacts.
- `figures/` — finalized figure images produced from notebooks (not always auto-generated by CI).

## Quick start (recommended)

1. Create environment (conda, recommended):

    ```bash
    conda env create -f environment.yml
    conda activate habits-1
    pip install -r requirements.txt  # if you prefer pip-only
    ```

2. Run all notebooks and generate executed notebooks and figures:

    ```bash
    make figures
    ```

    Notes:
    - `make figures` calls `scripts/make_figures.sh`, which finds notebooks under
       `notebooks/`, executes each notebook from its directory (so relative
       `../data/...` paths work), and places executed notebooks in
       `notebooks/executed/`.
    - The script uses `papermill` if available, otherwise falls back to
       `nbconvert --execute`.

## Data layout and how code expects it

- `data/csvs/behavior.csv` — project-level behavioral summary table. Used to
   compute per-subject cohorts (patients vs controls) and to seed `SUBJECTS`.
- `data/csvs/rate.csv` — response-rate aggregates used by some analyses.
- `data/trial_info/` — per-subject trial CSVs named `R01_####-trial_info.csv`.
   - These files are read by `scripts.util.read_trial_info(subid)` and are
      expected to contain columns such as `run`, `phase`, `event`, `resp`, `corrResp`, `wasRnf`, `globalClock_t`, `devalued_coin`, `selected_coin`, `coin_img`, etc.
   - `scripts/util.py` computes absolute `PROJECT_DIRECTORY` based on the file
      location so functions work regardless of current working directory.

## Key scripts and functions

- `scripts/util.py` (recommended reading):
   - Exposes `read_trial_info(subid)`, `get_consumption_score(subid)`,
      `get_devaluation_ratio(subid)`, `get_block_rate(subid, runid, blockid)`,
      `get_run_rate(subid, runid)`, and other convenience functions used by
      notebooks and analyses.
   - Loads `behavior.csv` and `rate.csv` at import-time and defines lists
      `SUBJECTS`, `HEALTHY_SUBJECTS`, and `PATIENTS` for convenience.
   - Use these helpers inside notebooks to avoid duplicating low-level I/O logic.
- `scripts/make_figures.sh`:
   - Exports the project root into `PYTHONPATH` so notebooks can `import scripts`.
   - Executes notebooks from their directory so relative `../data/...` paths
      inside notebooks resolve correctly. Outputs executed notebooks to
      `notebooks/executed/`.

## Notebooks

- The `notebooks/` folder contains cleaned figure notebooks (e.g., `figure2.ipynb`, `figure3.ipynb`, `figure4.ipynb`).
- Executed copies are written to `notebooks/executed/` by `scripts/make_figures.sh`.
- If a notebook references `../data/...` paths, the execution script runs it
   from the notebook directory; running a notebook from a different CWD can
   produce FileNotFoundError for data files.

## Environment and dependencies

- `environment.yml` contains a conda environment specification (`python=3.10`) with core packages and conda-forge channel.
- `requirements.txt` lists pip-installable packages required for notebooks and scripting (Jupyter, numpy, pandas, nilearn, nibabel, joblib, etc.).

## Troubleshooting notes

- ModuleNotFoundError for `numpy` / `pandas`, etc.: ensure you activated the `habits-1` conda env or installed `requirements.txt` into your environment.
- FileNotFoundError for `../data/csvs/behavior.csv`: ensure you run notebooks from their directory or use the provided `scripts/make_figures.sh` which handles that detail.
- If notebooks fail due to long-running cells, increase `--ExecutePreprocessor.timeout` in the script or run the notebook interactively.
- If you changed `scripts/util.py` and see different `PROJECT_DIRECTORY` behavior, you can revert to relative paths, but absolute-root logic is more robust for automated execution.

## Reproducibility suggestions and next steps

- CI: Add a lightweight CI job that creates the conda env (or uses pip), runs `scripts/make_figures.sh` for a smoke-test, and stores `notebooks/executed/` as artifacts.
- Packaging: Consider adding a `requirements-dev.txt` with testing tools and a `python -m pip install -e .` setup if you want importable package behavior.
- Data provenance: If allowed, add a `data/README.md` describing how raw trial CSVs were created and any pre-processing steps.
